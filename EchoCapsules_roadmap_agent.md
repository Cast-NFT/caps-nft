# EchoCapsules — Project Roadmap & Agent Brief

> Working project name: **EchoCapsules**  
> Concept type: NFT Time Capsule Collection  
> Core loop: **Mint Mystery Box → Reveal Capsule + Rarity → Phase 2 Market Hold → Burn/Combine → Inner Archive later**

---

## 0. Project Name

### Recommended Name

# EchoCapsules

### Why this name fits

**EchoCapsules** fits the project better than a generic name like “Time Capsule NFT” because the collection is not only about storing a message. It also has:

- sealed capsules,
- rarity reveal,
- market trading,
- burn-combine mechanics,
- inherited history from burned NFTs,
- and a later inner message layer.

The word **Echo** connects directly to the project mechanic called **Echo Inheritance**, where forged Capsules carry traces from Capsules burned before them.

### Naming Positioning

Use:

- **EchoCapsules** as the main collection/project name.
- **The Inner Archive** as the hidden message layer.
- **The Furnace** as the burn-combine system.
- **Echo Inheritance** as the history/parent-trace system.

### Avoid using as main name

Avoid using **Time Capsule NFT** as the final brand name because it sounds descriptive and generic. It is better used as the category/concept, not the project name.

Example:

> EchoCapsules is a Time Capsule NFT collection built around sealed rarity, market movement, forging, and personal messages.

## 1. Summary From Previous Discussion

This project started from a general NFT idea where holders would mint something that later reveals into random images.

The early direction was connected to CastMiner, Farcaster, mining, and cast-based activity. That direction was intentionally abandoned. The new project should not mention CastMiner or rely on Farcaster/cast mechanics.

The direction was reset into a clean project concept:

> A Time Capsule NFT where holders first mint a sealed Mystery Box, later reveal a Capsule with rarity, trade it during a market-focused phase, and eventually interact with a deeper message layer.

The chosen concept was based on **Idea A: Time Capsule**.

The emotional idea behind it:

> A Capsule is not just a delayed message. It is a sealed object that can hold something personal, expressive, or emotional, while still functioning as a collectible NFT with rarity and market mechanics.

The project should balance two sides:

1. **Collectible / market side** — rarity, traits, OpenSea trading, burn-combine, floor discovery, holder strategy.
2. **Personal / emotional side** — the inner message layer that will be opened later.

The final locked content model is:

# Hybrid Model

- Art / visual shell comes from the project.
- Personal message comes from the user later.
- Rarity and traits drive market value first.
- The message layer becomes the deeper value layer later.

EchoCapsules is a sealed NFT capsule collection.

Every NFT begins as a **Mystery Box**.

After the mint closes, each Mystery Box reveals into a **Capsule** with:

- rarity tier,
- visual shell,
- traits,
- market identity,
- and forge potential.

However, the Capsule is not fully open after the rarity reveal.

It has two layers:

## 2.1 Visual Layer

The visual layer is created by the project.

This layer includes:

- Capsule art,
- tier,
- traits,
- shell design,
- rarity score,
- market position,

This layer is revealed in Phase 2.

## 2.2 Inner Layer

The inner layer is connected to the personal message system.

This layer is not rushed.

It remains sealed during Phase 2 so the collection can first build identity, trading activity, and market structure around visible Capsules.

The inner layer should be mentioned in the roadmap, but not pushed as an immediate feature.

Correct wording:

> The inner layer remains sealed.

Avoid wording like:

- Coming soon
- TBA
- Future utility
- Phase 3 soon
- Content reveal soon
- We are still building this

## 3. Main Project Flow

## Phase 1 — Mystery Box Mint

Users mint a sealed Mystery Box.

At this stage:

- Capsule type is hidden.
- Rarity tier is hidden.
- Visual shell is hidden.
- Forge value is unknown.
- Traits are not visible.
- Market value is based on speculation.

Purpose:

- create initial mystery,
- make every minter start from the same unknown point,
- and prepare the collection for the first reveal event.

---

## Phase 2 — Capsule Reveal

After the mint closes, every Mystery Box opens into a Capsule.

This is the first major reveal.

Each Capsule reveals:

- rarity tier,
- visual shell,
- Capsule identity,
- trait set,
- market position,
- and forge potential.

Important:

> The inner message layer is still sealed.

This phase is designed to create the first real market split.

Before Phase 2, every Box is unknown.

After Phase 2, collectors can price NFTs differently based on:

- rarity,
- visible traits,
- tier strength,
- art quality,
- set-building value,
- and burn-combine potential.

---

## Phase 2 Hold Period — Main Market Stage

After Capsule Reveal, the project enters a Phase 2 hold period.

This is not an empty waiting period.

This is the main market-building stage.

The goal is to let OpenSea volume, floor discovery, holder strategy, and burn-combine activity develop before the inner layer opens.

During this period:

- Capsules stay visually revealed.
- Rarity and traits stay visible.
- The inner layer stays sealed.
- Burn-combine is active.
- Holders can trade, sweep, build sets, and forge higher forms.

The project should focus on Phase 2 first.

Do not rush Phase 3.

Phase 3 should exist in the roadmap only as the deeper layer of the project, not as an immediate event.

## Phase 3 — Inner Archive

Phase 3 is the deeper message layer.

It should not be the current focus.

The roadmap may mention it as **The Inner Archive**, but it should not use language that makes the project look unfinished.

> Phase 2 reveals the shell.

## 4. Rarity System

Use traditional NFT rarity distribution.

The previous idea of inverted rarity was rejected because it would confuse the market.

Final rarity structure:

| Tier | Supply Share | Role |
|---|---:|---|
| Common | 60% | Main supply, highest liquidity, primary forge fuel |
| Rare | 25% | Mid-tier collector target |
| Epic | 12% | Scarcer tier with stronger market value |
| Legendary | 3% | Premium natural tier |
| Mythic | <0.3% | Ultra rare, mostly created through forging |

## 4.1 Important Rarity Principle

Common should not be useless.

Common has the highest supply, but it must also become the main fuel of the ecosystem.

This keeps the Common floor active because traders and forgers need Common Capsules for upgrades and skip-path forging.

## 5. Capsule Content Model

The selected model is:

# Hybrid

This means:

- Project provides the Capsule art.
- User provides the message later.
- The Capsule can be traded before the message layer opens.
- Market value comes first from rarity, traits, art, and forge potential.
- Emotional value comes later from the inner message layer.

## Why Hybrid Works

Pure user-written NFTs are personal but may be weak as collectibles.

Pure pre-curated NFTs are easier for trading but less emotionally unique.

Hybrid gives both:

- collectible value,
- visual rarity,
- OpenSea market behavior,
- and later emotional depth.

## 6. Phase 2 Market Design

Phase 2 is built to support market activity before the Inner Archive opens.

This stage should create multiple reasons for holders to trade.

## 6.1 Tier Trading

Each tier should develop its own market:

- Common floor,
- Rare floor,
- Epic floor,
- Legendary floor,
- Mythic floor.

Collectors should be able to compare tiers clearly.

## 6.2 Trait Hunting

Rarity should not only come from tier.

Each Capsule should also have sub-traits.

The conversation already locked that designs/sub-traits exist.

Possible trait categories:

- Capsule Type,
- Shell Material,
- Seal Type,
- Aura,
- Age Mark,
- Background,
- Core Symbol,
- Echo Strength,
- Forge Potential,
- Archive Mark,
- Frame Style,
- Color Palette,
- Glyph/Symbol.

Why this matters:

If tier is the only rarity signal, market behavior becomes too simple.

If sub-traits exist, even a Common can become desirable if it has a rare trait combination.

This creates more sniping, bidding, and trading activity.

## 6.3 Visible Art in Phase 2

During Phase 2, the Capsule art should already be visible.

Only the inner message layer remains sealed.

This is important because Phase 2 needs the Capsule to be desirable as a standalone collectible.

The message layer should be treated as a second value layer, not the only reason to care.

## 7. Burn-Combine System — The Furnace

The burn-combine system is called:

# The Furnace

The Furnace is active during Phase 2.

This is important because Phase 3 is being held back, so the project needs an active mechanic during the market stage.

The Furnace allows holders to burn Capsules and create higher-tier Capsules.

This creates:

- supply reduction,
- buying pressure,
- tier upgrades,
- set-building demand,
- Mythic hunting,
- and new market events.

---

## 8. Burn-Combine Ratios

Final selected structure:

| Burn Input | Output | Path Type |
|---|---|---|
| 3 Common | 1 Rare | Step path |
| 2 Rare | 1 Epic | Step path |
| 7 Common | 1 Epic | Premium skip path |
| 2 Epic | 1 Legendary | Step path |
| 15 Common | 1 Legendary | Premium skip path |
| 2 Legendary | 1 Mythic | Step path |
| 1 Common + 1 Rare + 1 Epic + 1 Legendary | 1 Mythic | Rainbow set path |

## 8.1 Why These Ratios Make Sense

The original idea allowed Common to skip directly into Epic and Legendary at very cheap ratios.

That would make Common extremely powerful, but it could weaken the value of Rare and Epic.

The final version keeps skip paths, but makes them more expensive.

This creates two options:

### Step-by-step path

More efficient, but requires specific tiers.

Example:

- 3 Common → 1 Rare
- 2 Rare → 1 Epic
- 2 Epic → 1 Legendary

### Premium skip path

Uses Common directly, but costs more.

Example:

- 7 Common → 1 Epic
- 15 Common → 1 Legendary

This keeps Common valuable without destroying the purpose of Rare and Epic.

## 9. Echo Inheritance

When Capsules are burned inside the Furnace, their history should not disappear completely.

The forged Capsule carries traces from the burned Capsules.

This is called:

# Echo Inheritance

A forged Capsule can carry:

- parent Capsule references,
- burn history,
- previous tier origin,
- echo fragments,
- forge generation,
- creation path,
- and source tier records.

Example:

A naturally revealed Legendary is different from a Legendary forged from 2 Epic Capsules.

A Mythic forged from 2 Legendary Capsules is different from a Mythic forged through the rainbow set path.

This gives forged NFTs a story and collector identity.

## 10. Supply Mechanics

## 10.1 Starting Supply Example

If the total supply is 10,000:

| Tier | Percent | Estimated Supply |
|---|---:|---:|
| Common | 60% | 6,000 |
| Rare | 25% | 2,500 |
| Epic | 12% | 1,200 |
| Legendary | 3% | 300 |
| Mythic | <0.3% | Very limited / mostly forged |

Mythic should be extremely limited.

It should not feel like a normal tier.

## 10.2 Deflation Logic

Every Furnace action removes more NFTs than it creates.

Examples:

- 3 Common burn → 1 Rare minted = net -2 supply
- 2 Rare burn → 1 Epic minted = net -1 supply
- 7 Common burn → 1 Epic minted = net -6 supply
- 15 Common burn → 1 Legendary minted = net -14 supply
- 2 Legendary burn → 1 Mythic minted = net -1 supply
- Rainbow set burn 4 Capsules → 1 Mythic minted = net -3 supply

This creates deflation over time.

The more people forge, the smaller the circulating supply becomes.

## 10.3 Why Phase 2 Burn-Combine Matters

Since Phase 3 is not immediate, burn-combine must be active during Phase 2.

Without this mechanic, Phase 2 could become passive after the initial rarity reveal.

With the Furnace active, Phase 2 has ongoing activity:

- people buy lower tiers to forge,
- whales accumulate Common,
- set builders hunt combinations,
- forged NFTs create new listings,
- Mythic hunters generate attention,
- and supply slowly decreases.

## 12. Holder Strategy Paths

The project should make holders feel there are multiple ways to participate.

## 12.1 The Holder

Keeps the Capsule and waits.

This holder values future depth and collection identity.

## 12.2 The Trader

Trades based on tier, floor movement, rarity, and traits.

This holder helps create marketplace volume.

## 12.3 The Sweeper

Buys multiple lower-tier Capsules, especially Common.

This holder may control forge fuel supply.

## 12.4 The Forger

Uses the Furnace to create higher-tier Capsules.

This holder reduces supply and creates new market listings.

## 12.5 The Set Builder

Builds combinations such as:

- Common + Rare + Epic + Legendary,
- full Capsule type set,
- same Aura set,
- same Archive Mark set,
- same Material set.

## 12.6 The Mythic Hunter

Targets the rarest creation paths.

This holder may try to create Mythic Capsules through:

- 2 Legendary → 1 Mythic,
- or rainbow set → Mythic.

## 13. Whale Incentive System

Do not simply give free Mythics to top holders without control.

That can look unfair and may cause whales to farm the system.

Use a more balanced system:

# Echo Score

Echo Score is a Phase 2 leaderboard score based on healthy collection participation.

It should reward:

- holding,
- forging,
- set building,
- trading activity,
- and long-term participation.

It should not only reward raw wallet size.

## 13.1 Suggested Echo Score Formula

Echo Score can combine:

- number of Capsules held,
- tier weight,
- number of successful forges,
- complete sets held,
- holding duration,
- non-wash trading activity,
- and Mythic creation attempts.

## 13.2 Recommended Rewards

Instead of direct guaranteed Mythic allocation, use controlled incentives:

- top Echo Score holders get priority access to special forge windows,
- top forgers get profile placement in the Furnace history,
- top set builders get limited Archive Marks,
- top holders get whitelist for future archive mechanics,
- top trader/volume participants get non-supply-breaking cosmetic rewards.

Avoid giving too many free top-tier NFTs because it can weaken rarity trust.

## 13.3 Safer Mythic Incentive

If Mythic rewards are used, make them tied to forging, not just wallet size.

Example:

> Top Echo Score wallets receive a temporary reduced-fee Mythic forge window, but they still need to provide the required burn materials.

This rewards whales without printing free supply.

# 16. Full Public Roadmap

This is the version suitable for website, mint page, or public docs.

## Chapter I — Sealed Mint

Mystery Boxes are minted.

Each Box starts sealed.

At this stage, no Capsule type, rarity tier, visual shell, trait set, or forge value is visible yet.

Every holder begins from the same unknown position.

The first market forms around speculation: what the Box may become after the Capsule Reveal.

---

## Chapter II — Capsule Reveal

After the mint closes, every Mystery Box opens into a Capsule.

Each Capsule reveals its:

- rarity tier,
- visual identity,
- trait set,
- Capsule type,
- and forge potential.

The inner layer remains sealed.

This is where the market starts separating Capsules by tier, rarity score, visual shell, and collector demand.

---

## Chapter III — Open Market

Capsules enter the open market.

Holders can list, collect, sweep, trade, and build positions based on visible rarity and forge potential.

This stage is focused on market discovery.

Common becomes the main fuel.  
Rare and Epic become upgrade targets.  
Legendary becomes the premium natural tier.  
Mythic becomes the rarest forged form.

The inner layer is not touched during this stage.

---

## Chapter IV — The Furnace

The Furnace allows Capsules to be burned and combined into higher forms.

Burn paths:

- 3 Common → 1 Rare
- 2 Rare → 1 Epic
- 7 Common → 1 Epic
- 2 Epic → 1 Legendary
- 15 Common → 1 Legendary
- 2 Legendary → 1 Mythic
- 1 Common + 1 Rare + 1 Epic + 1 Legendary → 1 Mythic

Every forge removes Capsules from circulation.

Every higher form created through the Furnace has a visible origin path.

---

## Chapter V — Echo Inheritance

Forged Capsules carry traces from the Capsules used to create them.

This is Echo Inheritance.

A forged Capsule can carry parent references, generation data, forge path, and echo history.

This makes a forged Capsule different from a naturally revealed Capsule of the same tier.

The market can value not only what tier a Capsule is, but also how it was created.

---

## Chapter VI — The Inner Archive

A Capsule is more than its shell.

The visual layer reveals rarity.  
The inner layer holds the message.

During the market stage, the Inner Archive remains sealed.

The Capsule can be collected, traded, forged, or held before that layer is opened.
